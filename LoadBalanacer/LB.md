# ⚖️ Load Balancing

## 📖 What is Load Balancing?

**Load Balancing** is the process of **distributing incoming user requests or network traffic across multiple servers**.

Instead of sending all requests to a single server, a **Load Balancer** shares the traffic among multiple servers. This improves **performance**, **availability**, and **reliability**.

> **Simple Definition:**  
> **Load Balancing distributes incoming traffic across multiple servers to ensure better performance, high availability, and fault tolerance.**

---

## 🎯 Why Use Load Balancing?

### 📈 Scalability
- Easily handle more users by adding more servers.
- Helps applications grow without affecting performance.

### 🛡️ High Availability
- If one server fails, the Load Balancer automatically sends traffic to healthy servers.
- Ensures the application remains available.

### ⚡ Better Performance
- Distributes traffic evenly across servers.
- Prevents any single server from becoming overloaded.

### 🔧 Maintenance Flexibility
- Servers can be updated or maintained without affecting users.
- Traffic is automatically redirected to other healthy servers.

---

## 🌐 Types of AWS Load Balancers

AWS provides **4 types** of Elastic Load Balancers (ELB):

1. ⚖️ **Application Load Balancer (ALB)**
2. ⚡ **Network Load Balancer (NLB)**
3. 🛡️ **Gateway Load Balancer (GWLB)**
4. 📦 **Classic Load Balancer (CLB)**
